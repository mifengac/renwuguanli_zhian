# 任务管理系统新增模块汇报：监测提醒

---

## 一、业务介绍

### 1.1 模块背景

监测提醒模块是在原有任务管理系统基础上新建的专项工作跟踪与自动提醒功能。其核心定位是：**将周期性、重复性的专项工作事项纳入系统统一管理，通过自动生成执行记录、自动触发多渠道提醒，确保每一件事都有人跟、每一个节点不会漏**。

### 1.2 核心业务对象

监测提醒模块围绕三个层级的业务对象展开：

#### 第一层：监测方案

监测方案是整个模块的顶层载体，对应一类专项工作（如"季度报表上报"、"月度安全检查"等）。每个方案有明确的：

- **归口部门**：方案由哪个部门负责，权限管理以部门为边界
- **有效期**：方案的起始和结束日期，过期自动停止生成新任务
- **状态流转**：支持草稿 → 启用 → 暂停 → 已结束的全状态管理，随时可以调整

#### 第二层：监测项

监测项是方案下的具体事项，是系统执行的最小配置单元。每个监测项可以灵活配置：

- **执行周期**：一次性、每周、每月、每季度或自定义，适应不同频率的工作要求
- **截止时间**：精确到时分秒的截止时间点（如每月25日17:00）
- **完成要求**：可要求责任人上传附件和/或填写完成说明，确保完成质量可追踪
- **人员分工**：
  - **责任人（OWNER）**：对该事项的完成结果负责
  - **提醒对象（REMIND）**：在责任人外需要同步接收提醒的相关人员
  - **抄送人（CC）**：仅接收完成状态通知，无需操作
- **提醒规则**：每个监测项可配置多条提醒规则（详见1.3节）

#### 第三层：监测实例

监测实例是系统按照监测项的周期配置**自动生成**的执行记录，相当于"每一次周期任务的工单"。例如，一个"每月提交报告"的监测项，系统会自动为每个月生成一条实例记录，实例包含：

- 所属周期标签（如"2026年3月"）
- 本次截止时间
- 当前状态：**待完成 / 已完成 / 已逾期 / 已取消**
- 责任人的完成记录（完成人、完成时间、说明、附件）
- 提醒次数和最后提醒时间

### 1.3 提醒规则机制

提醒规则是本模块的核心技术特色，支持精细化、灵活化的提醒策略配置：

**触发时机**（三选一）：
- **到期前**：提前 N 天 N 小时发出提醒，给责任人充足的准备时间
- **到期当天**：在截止日当天指定时间发送提醒
- **逾期后**：超过截止时间仍未完成时，持续催办

**重复方式**（四选一）：
- **仅一次**：在计算出的触发时间点发送一次
- **每天一次**：按天重复发送，直到完成或达到最大发送次数
- **工作日每天**：仅在工作日发送，非工作日跳过
- **每 N 小时**：以N小时为间隔高频提醒，适用于紧急事项

**其他控制参数**：
- 最大发送次数：防止无限次轰炸
- 完成后停止：任务完成后自动停止后续提醒，避免打扰

**通知渠道**（可叠加）：
- **短信提醒**：通过对接企业短信平台（Oracle短信队列），直接发至责任人手机，确保不遗漏
- **系统内提醒**：在系统界面内推送通知，便于集中查看和跟踪

提醒内容支持模板变量，自动填充方案名称、事项名称、所属周期、截止时间等关键信息，无需人工编写。

### 1.4 系统功能页面

模块提供三个核心页面：

| 页面 | 说明 |
|------|------|
| **专项工作** | 管理监测方案与监测项，配置周期、人员和提醒规则 |
| **实例管理** | 查看所有周期实例的执行现状，完成操作、查看逾期情况 |
| **提醒日志** | 查询每一条提醒的发送明细，包括发送状态、通道、接收人、内容等 |

## 二、解决的问题

### 2.1 专项工作"靠人盯"，节点容易遗漏

过去，专项工作的重要节点（如报表上报、检查完成等）主要依赖相关人员自行记忆或通过微信、电话等人工方式提醒，没有系统记录，责任边界模糊。一旦负责人工作繁忙或调岗，极易造成节点遗漏。

**监测提醒模块的解决方式**：将专项工作事项录入系统，明确责任人、截止时间和提醒频率，由系统自动提醒，彻底告别"靠记忆、靠人盯"的被动工作方式。

### 2.2 周期性工作重复创建，占用大量管理精力

对于每周、每月、每季度的固定事项，以往需要管理人员周期性地逐条新建任务，操作繁琐，且容易因疏漏导致某一周期的跟踪缺失。

**监测提醒模块的解决方式**：配置一次监测项，系统按照设定周期**自动超前生成实例**（默认提前60天生成），无需人工反复创建，周期任务管理从"手动操作"升级为"配置即运行"。

### 2.3 提醒不及时，逾期无人催办

传统工作模式下，任务临近截止日缺乏有效预警机制，逾期后也缺乏自动催办，需要管理人员人工逐一跟进，工作量大且效果参差。

**监测提醒模块的解决方式**：
- 支持"到期前 N 天"提前预警，争取处理时间
- 任务逾期后自动触发"逾期后"规则，持续催办，系统自动处理追踪，无需人工介入
- 短信+系统双渠道并行，确保提醒不被遗漏

### 2.4 执行情况不透明，管理层难以掌握全貌

以往缺少专项工作执行情况的统一视图，管理层无法快速了解哪些事项已完成、哪些逾期、哪些即将到期，只能依赖逐级汇报，信息滞后。

**监测提醒模块的解决方式**：实例管理页面提供全量实例的状态视图，支持按方案、状态、周期等维度筛查，管理层可随时查看各部门专项工作的完成现状，做到"心里有数"。

### 2.5 责任划分不清，出问题无法追溯

当专项工作未按时完成时，往往说不清楚到底谁收到了提醒、谁没有按时处理，也无法确认提醒是否真正发出过。

**监测提醒模块的解决方式**：
- **提醒日志**：记录每一条提醒的发送时间、接收人、通道、内容和发送状态，做到有据可查
- **操作日志**：记录方案配置、实例完成、重新开启等所有操作的操作人和时间，全程留痕
- 人员角色明确（责任人 / 提醒对象 / 抄送人），提醒到人、责任到人

## 三、提高效能

### 3.1 自动化运转，大幅降低管理成本

监测提醒模块实现了"配置一次、长期运转"的工作模式。管理员只需在系统中配置好方案、监测项和提醒规则，后续每个周期的实例生成、提醒发送全部由系统自动完成，管理人员从繁琐的手动提醒工作中解放出来，专注于更高价值的工作。

**估算效益**：若一个部门管理10项周期性专项工作，每月人工提醒创建和跟进需耗费约2~3个工作日，推广后预计每月可节省80%以上的此类事务性工作量。

### 3.2 精准提醒，提升事项按时完成率

系统支持多阶段、多频次提醒策略：

- **三阶段覆盖**：到期前预警 → 到期当天提示 → 逾期催办，形成完整提醒闭环
- **渐进式提醒**：可配置随着接近截止日逐步加大提醒频率（如：提前3天每天一次 + 到期当天每2小时一次）
- **短信穿透**：短信提醒不依赖责任人是否登录系统，极大提升提醒到达率

通过精准、及时的多层次提醒，有效降低"忘记、没看到"造成的逾期比例。

### 3.3 信息透明，管理决策更高效

- **实时状态看板**：责任人可随时看到自己名下的待完成实例和逾期实例，优先级一目了然
- **部门级视图**：部门管理员可查看本部门所有监测项的完成情况，实现对进度的主动掌控
- **超级管理员视图**：全局可见，跨部门的专项工作完成情况集中呈现，支撑整体协调和决策

### 3.4 提醒记录可查，责任落实有据

提醒日志模块完整记录了每次提醒的发送与送达情况，回答了以下关键管理问题：
- 这个事项提醒了几次？
- 每次提醒是否成功送达？
- 提醒发给了谁？
- 内容是什么？

当出现工作延误或责任纠纷时，管理人员可通过日志快速还原完整链条，有效避免"踢皮球"的情况，推动责任文化落地。

### 3.5 灵活配置，适配多类型业务场景

模块设计充分考虑了实际业务的多样性：

| 业务场景 | 对应配置 |
|----------|----------|
| 一次性核查任务 | 周期类型选"一次性"，指定固定截止日 |
| 每月报表上报 | 周期类型选"每月"，指定每月几号17:00截止 |
| 每周例会纪要提交 | 周期类型选"每周"，指定星期几截止 |
| 季度合规检查 | 周期类型选"每季度" |
| 重大节点紧急催办 | 提醒规则设置"逾期后每2小时"，短信+系统双渠道 |
| 无须短信打扰的日常小事 | 仅开启系统内提醒，关闭短信 |

---

## 总结

监测提醒模块从"专项工作事项如何不遗漏、不逾期"这一核心管理痛点出发，构建了**方案 → 监测项 → 监测实例**的三层业务结构，配合**灵活的提醒规则**和**短信+系统双渠道推送**，实现了专项工作的全生命周期系统化管理。模块上线后，将显著降低管理人员的事务性工作负担，提升专项工作的按时完成率，同时通过完整的操作和提醒日志，让管理有据可依、责任落实有迹可查，为基础管控中心专项工作管理提质增效。
